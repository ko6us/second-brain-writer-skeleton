# Second Brain Writer Skeleton

AIエージェント駆動型の「第二の脳」構築と執筆ワークフローのためのスケルトンリポジトリ。
モジュラー（アドオン）構造を採用しており、純粋な知識管理から、特定プラットフォーム向けの執筆や専門的な分析まで、柔軟に拡張可能です。

## 🚀 コンセプト
このリポジトリは、日々のインプット（知識）を蓄積し、そこから自分独自の思想（スタンス）を抽出し、一貫性のあるアウトプット（記事など）を生み出すためのワークスペースです。

AIエージェント（Gemini CLI / Antigravity等）と協調することを前提に設計されており、インプットからアウトプットまでの「思想のライフサイクル」を回すための構造を提供します。

## 📂 ディレクトリ構成
- `knowledge/`: あなたの「第二の脳」。あらゆる知識とスタンスのコア蓄積場所。
- `extensions/`: 特定の用途に特化するためのアドオン（例：note執筆、投資分析）。
- `.agents/skills/`: 第二の脳を育てるための共通のAIカスタムスキル。
- `GEMINI.md`: AIエージェントへの基本指示書（プロジェクトの憲法）。

## 🧩 モジュラーアーキテクチャ（拡張機能）
このリポジトリの強みは、**「知能の切り替え」**ができる点にあります。
ルートの `GEMINI.md` は純粋な知識の蓄積と構造化（Core）に特化しています。

記事を書いたり、特定の分析を行いたい場合は、`extensions/` 配下の専用の `GEMINI.md` をAIに読み込ませることで、AIをそのタスクのプロフェッショナル・モードに切り替えることができます。

*例：note記事を書く場合*
`extensions/note-article/` には、note執筆用の専用スキル（publishなど）やGist連携スクリプトが隔離されています。

## 🛠 セットアップ
1. このリポジトリをテンプレートとして使用、またはクローンします。
2. Obsidian等のMarkdownエディタで開き、`knowledge/raw/` にメモを放り込み始めます。
3. AIエージェントを起動し、ルートの `GEMINI.md` を読み込ませることで、「ナレッジマネージャー」として機能し始めます。

詳細な初期設定については [SETUP.md](SETUP.md) を参照してください。
各フォルダの詳しい使い方は [FOLDER_GUIDE.md](FOLDER_GUIDE.md) を参照してください。
