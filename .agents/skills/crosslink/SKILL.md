# Skill: crosslink

## 目的
複数の記事を横断的に分析し、共通するパターン・概念・主張を発見して概念ノートとして抽出する。ナレッジベースの「複利効果」を生む中核スキル。

## トリガー
ユーザーが「記事間の共通点を探して」「パターンを抽出して」「ナレッジを整理して」などと指示したとき。

## 手順

### 1. 対象の特定
- ユーザーが指定した記事群、またはタグ・テーマで絞り込んだ記事群を読む
- 各記事の `📋 執筆メモ` と `🔗 参照知識` もあわせて確認する

### 2. パターンの抽出
以下の観点で横断分析する：
- **繰り返し登場する概念**: 複数記事で同じ考え方が別の言葉で表現されていないか
- **共通する主張・スタンス**: 記事をまたいで一貫して言っていること → `stances/` 候補
- **再利用可能なフレームワーク**: 特定の記事で使った思考法が他でも使えそう → `frameworks/` 候補
- **矛盾・不整合**: 記事間で異なることを言っている箇所

### 3. 概念ノートの作成提案
発見したパターンごとに：
- どの `concepts/` サブフォルダに置くべきか
- ノートのタイトル案
- 関連する記事リスト

をユーザーに提示し、承認後にノートを作成する。

### 4. 既存ノートの更新
- 新たに作成したノートから既存記事への `[[リンク]]` を設定する
- 関連する既存記事の `🔗 参照知識` セクションにリンクを追加する提案をする

### 5. 報告
```markdown
## 横断分析結果

### 発見したパターン
1. **{パターン名}** - {記事A}, {記事B}, {記事C} に共通
   → 提案: `concepts/{フォルダ}/{ファイル名}.md` として作成

### 発見した矛盾
- {記事X} と {記事Y} で「○○」の定義が異なる

### 推奨アクション
- ...
```
