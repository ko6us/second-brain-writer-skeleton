# Skill: lint

## 目的
`scripts/audit.py` を使用してナレッジベース全体の健全性をチェックし、リンク切れ、タグの不整合、孤立ノート、スタンスとの矛盾などを検出し、修正を支援する。

## ワークフロー

1.  **監査の実行**:
    - `python scripts/audit.py` を実行する。
    - 生成された最新のレポート `scratch/link-audit-YYYY-MM-DD.md` を読み込む。

2.  **結果の分析**:
    - レポートの「🔴 優先度高」「❌ デッドリンク」「🏝️ 孤立ノート」を優先的に確認する。
    - 特にスタンス（stances）へのリンク漏れ（Pattern C）や記事の参照知識漏れ（Pattern D）を重視する。

3.  **提案と修正**:
    - ユーザーに対し、検出された問題のサマリーを報告する。
    - 自動修正が可能なもの（リンクの追記など）については、一括修正の提案を行う。
    - 承認後、各ファイルの `## 🔗 参照知識` セクションや本文にリンクを追記する。

4.  **事後検証**:
    - 修正後、再度 `python scripts/audit.py` を実行し、問題が解消されたことを確認する。

## 推奨実行タイミング
- ユーザーが「lintして」「ヘルスチェックして」「リンク切れを確認して」と指示したとき。
- 大量のノート（10件以上）を整理、またはマージした直後。

## 注意事項
- `audit.py` の実行には `PyYAML` ライブラリが必要。
- リンクの追加は、既存の文章の文脈を壊さないように慎重に行うこと。
- デッドリンクの修正では、単純なタイポか、ファイルが未作成（backlog）かを判断し、必要に応じてリンクを外すかファイル作成を提案する。
