# Skill: review

## 目的
記事ドラフトを `knowledge/` の概念ノートおよびWeb情報と照合し、修正すべき点を報告する。Phase C（修正サイクル）の中核スキル。

## トリガー
ユーザーが「この記事をレビューして」「チェックして」などと指示したとき。

## 手順

### 1. 記事の読み取り
- 指定された記事ファイルを読む
- `📋 執筆メモ` からターゲット読者・トーン・意図を把握する
- `🔗 参照知識` に記載された概念ノートを全て読む

### 2. 整合性チェック
**執筆ガイドとの照合:**
- `knowledge/concepts/frameworks/note執筆ガイド.md` のトーン・表現ルール・禁止事項に違反していないか確認

**概念ノートとの照合:**
- 参照先の概念ノートに表現ルール・用語定義がある場合、記事がそれに従っているか確認
- `stances/` 内の主張定義書と矛盾していないか確認
- `frameworks/` 内のフレームワーク定義と異なる記述がないか確認

**記事構成の確認:**
- 論理の飛躍がないか
- 主張と根拠の対応関係が明確か
- ターゲット読者に対して難易度が適切か

### 3. ファクトチェック（必要に応じて）
- 統計データ・数値の引用がある場合、Tavily MCPでWeb検索して正確性を検証する
- 検証結果を報告し、誤りがあれば修正案を提示する
- 検証済み情報は `concepts/memos/` にファクトノートとして保存を提案する

### 4. レビュー報告
以下の形式でユーザーに報告する：

```markdown
## レビュー結果: {記事タイトル}

### 🔴 要修正（概念ノートとの矛盾・事実誤認）
- ...

### 🟡 推奨修正（構成・表現の改善提案）
- ...

### 🟢 確認済み（問題なし）
- ...

### 📝 ファクトチェック結果
- ...
```

**重要**: 修正はユーザー承認後に行う。勝手に書き換えない。
