# Skill: ingest

## 目的
`knowledge/raw/` に置かれた生素材を読み取り、適切な `concepts/` サブフォルダに構造化された知識ノートを作成する。

## トリガー
ユーザーが「これを整理して」「ingestして」「ノートにまとめて」などと指示したとき。

## 手順

### 1. 素材の読み取り
- 指定されたファイル（またはraw/内の未処理ファイル）を読む
- 素材のタイプを判定する：
  - アイデアメモ → frameworks/ または stances/ 候補
  - Perplexityチャット履歴 → memos/ 候補
  - 外部記事・読書メモ → memos/ 候補
  - 連載・シリーズ構想 → series/ 候補

### 2. 分類先の決定
以下の基準で保存先フォルダを判断する：

| フォルダ | 判断基準 |
|---|---|
| `frameworks/` | 複数記事で繰り返し参照される手法・フレームワークの定義 |
| `series/` | 複数回にわたる連載の構成・公開順序・ルール |
| `stances/` | ぶれてはいけない自分の主張・立場 |
| `memos/` | 外部知識の整理・読書メモ・チャット履歴の構造化 |

判断に迷う場合はユーザーに確認する。

### 3. ノートの作成
- **ファイル名**: 内容を端的に表す日本語名（例：`AI時代の働き方.md`）
- **構成**:
  ```markdown
  # ノートタイトル

  ## 要約
  <!-- 3行以内のサマリー -->

  ## 主要な概念・知見
  <!-- 箇条書きで整理 -->

  ## 記事への活用ポイント
  <!-- このノートを記事執筆にどう使えるか -->

  ## 出典
  <!-- 元素材のファイルパスや参照URL -->
  ```
- `[[リンク]]` 形式で関連する既存ノートへのリンクを含める
- 既存の概念ノートと関連がある場合、既存ノート側にも相互リンクを追加する

### 4. 完了報告
- 作成したノートのパスと要約を報告する
- raw/ の元素材は削除せず残す（ソースオブトゥルースとして保持）
