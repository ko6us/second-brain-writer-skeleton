# Align Skill

## 概要
執筆中の記事（ドラフト）が、既存の「スタンスノート」や「フレームワーク定義」と論理的に矛盾していないか、思想がブレていないかを厳格にチェックします。

## 対応フェーズ
- **Phase C (修正サイクル)**: 記事の質を「思想」のレベルで担保する。

## 実行手順

### 1. 文脈の把握
- 対象となる記事ファイルを読み込みます。
- 記事の `🔗 参照知識` セクション、または記事のテーマから、関連するスタンスノートを特定し、読み込みます。

### 2. 整合性スキャン
- 以下の観点で矛盾を探します：
    - **定義の不一致**: スタンスノートで「A」と定義しているものを、記事で「B」として扱っていないか。
    - **優先順位の逆転**: スタンスノートで「人間が決定する」としているのに、記事で「AIに任せる」といった記述がないか。
    - **トーンの乖離**: スタンスノートで指定された「読者への寄り添い」などの態度が守られているか。

### 3. 指略と提案
- 矛盾が見つかった場合、以下の形式で報告します：
    - **【警告】**: 矛盾している箇所（引用）
    - **【根拠】**: 参照したスタンスノートの記述
    - **【修正案】**: スタンスに合わせるための書き換え案

## 成功の定義
- 複数の記事を読んだ読者が、一貫した「著者の人格・思想」を感じられる状態を維持できること。
- AIが勝手に論旨を変えてしまうリスク（AIの漂流）を防止すること。
