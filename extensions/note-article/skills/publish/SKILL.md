# Skill: publish

## 目的
記事をnote.comへの投稿に適した状態に仕上げる。Phase D（仕上げ・公開）のスキル。

## トリガー
ユーザーが「公開準備して」「noteに出せる状態にして」「publish」などと指示したとき。

## 手順

### 1. 表の変換
記事内にMarkdown表がある場合：
- **Gist化**: `python scripts/md-to-gist.py articles/{フォルダ名}/{記事}.md` を実行
- **画像化**: `python scripts/md-to-img.py articles/{フォルダ名}` を実行
- どちらを使うかはユーザーの指示に従う（デフォルトはGist化）

### 2. note.com向け整形
以下の調整を行う：
- **改行**: noteでは1行空けが段落区切りになるため、適切に空行を挿入
- **見出しレベル**: noteでは `##` が大見出し、`###` が小見出しとして機能する
- **箇条書き**: 長い箇条書きは適宜文章に変換する
- **リンク**: `[[内部リンク]]` はnoteでは機能しないため削除またはプレーンテキストに変換
- **画像**: ローカルパスの画像参照があれば警告する（noteにはアップロードが必要）

### 3. 読み直しチェック
- 冒頭3行で読者の興味を引けているか
- 末尾にCTA（次の記事への誘導、フォロー促進）があるか
- 有料ライン（ある場合）の位置が適切か

### 4. メタ情報の更新
- `status: draft` → 変更しない（公開後にユーザーが `published` に更新）
- 表をGist化した場合、`gist_tables` にURLを追加

### 5. 公開用テキストの出力
- 整形済みの本文を表示し、ユーザーがそのままnoteにコピペできる状態にする
- `📋 執筆メモ` と `🔗 参照知識` セクションは出力から除外する
